Connectors

Sync detection rules from remote Rulezet instances or compatible platforms. Pulls are triggered by admins.

What is a connector?

A connector links this instance to another Rulezet instance. Once configured, you can trigger a pull to import detection rules from the remote instance into this one.

  • Shadow user — a ghost account owns imported rules, keeping your account clean.
  • Become owner — you own all imported rules and bundles.
  • The original rule source (e.g. GitHub URL) is always preserved.
  • Rules store which instance they were synced from, even if the connector is later deleted.
Soft pull — safe import
  • Fetches all rules from the remote.
  • For each rule, checks locally by UUID then by content.
  • If a match is found → skip, nothing is changed.
  • If no match → the rule is added.
Existing rules, edits, and history are never affected.
Hard pull — force resync
  • Fetches all rules from the remote.
  • For each rule, checks locally by UUID then by content.
  • If a match is found → the local rule is moved to trash and replaced by the remote version.
  • If no match → the rule is added.
  • The local rule's update history is preserved and transferred to the new rule.
Use when the remote is authoritative and you want a full resync. Replaced rules can be restored from trash.
